# Core Thesis

Most reflection tools ask users to write freely or depend on AI-generated responses.

This system takes a different path.

It encodes reflection intelligence directly into a deterministic tree using:

- fixed-choice questions  
- decision nodes  
- psychological signals  
- branching logic  
- reflection prompts  
- dynamic summaries

The intelligence lives in the structure.

---

# Why This Matters

Reflection should be:

- predictable  
- repeatable  
- explainable  
- emotionally safe

A user should receive the same quality reflection every time.

That is difficult with generative AI.

It is natural with a well-designed tree.

---

# Three Psychological Axes

## Axis 1 — Agency

**Victim ↔ Victor**

Did the day happen *to me*, or did I still exercise choice within it?

Measures:

- ownership  
- response control  
- internal vs external locus

---

## Axis 2 — Contribution

**Entitlement ↔ Contribution**

Did I mainly track what I received, or what I gave?

Measures:

- recognition dependence  
- discretionary effort  
- value orientation

---

## Axis 3 — Radius

**Self ↔ Others**

Did my perspective remain narrow, or widen toward team, colleague, or customer?

Measures:

- self-focus under pressure  
- perspective taking  
- meaning through contribution

---

# Features

- JSON-driven architecture  
- Explicit decision nodes  
- Deterministic routing engine  
- Signal tallying across all three axes  
- Dynamic summaries based on patterns  
- Python CLI implementation  
- Human-readable tree data  
- Easy to extend and audit
